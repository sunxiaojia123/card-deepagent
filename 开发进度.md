- ToolMessage 缩短验证：card data 不在 LLM context，`artifact` 持久化到 checkpoint
- 前端：card 消息渲染为蓝色边框 JSON，历史加载时从 `artifact` 恢复

## Phase 3 架构总览

```
用户输入 → Agent 匹配 Skill → LLM 调用 call_internal_api(api_name, params)
  → 加载 apis.yaml 查找 schema → 校验参数
  → HTTP 请求 Mock API (localhost:9000)
  → 解析 {code, message, data}
  → TradingOutputMiddleware:
      code=0 + card → stream_writer("custom", {event:"card", data})
      ToolMessage(summary, artifact=data)
  → SSE stream: messages→text, updates→tool, custom→card
  → 前端渲染: text=逐字显示, tool=齿轮图标, card=蓝色JSON框
```

## 踩坑记录

| 问题 | 解决 |
|------|------|
| Postgres checkpointer 的 `aget_tuple()` 无法加载 messages（blob 版本不匹配） | 改用 `agent.aget_state(config)` 读取历史 |
| psycopg 默认不 autocommit，`init_db()` CREATE TABLE 被回滚 | `conn.set_autocommit(True)` |
| `astream` 单 stream_mode 返回 2-tuple `(mode, data)`，非文档中的 3-tuple `(namespace, mode, data)` | 适配器按 `(mode, data)` 解析 |
